# Vibe Coding – AI-Native Software Development

**קהל יעד:** תעשייה וניהול
**משך:** 13 שבועות
**היקף:** 3 שעות אקדמיות למפגש (מפגש אחד רציף, ללא רכיב "תרגול" נפרד בתכנית הלימודים)

## מבנה המפגש

הקורס מוכר במערכת כ"הרצאה" בלבד (3 ש"א), ללא שעת תרגול נפרדת. בהתאם לכך, כל מפגש הוא **יחידה רציפה אחת**:

* **90 דק' – תוכן והדגמה.**
* **45 דק' – תרגול מודרך, כחלק בלתי נפרד מאותו מפגש.**

## עקרון הצמצום: מה נכנס להרצאה ומה עובר לקריאה עצמאית

בסבב קודם זוהה שכמה שבועות היו עמוסים מדי לשקף 90 דקות בעומק אמיתי (בעיקר שבועות 2, 6, 9, 11, 12 — כל אחד הכיל 6-11 נושאים נפרדים). לכן בכל שבוע כאן:

* **הרצאה (90 דק') מכילה רק 3-4 נושאי-ליבה**, עם חלוקת זמן משוערת לכל נושא, כך שבאמת אפשר ללמד אותם בעומק ולא רק להזכיר אותם.
* **נושאים חשובים אך לא קריטיים ל"מה חייבים כדי להמשיך בפרויקט"** עברו לסעיף "קריאה עצמאית" — אלו נכנסים ל-`lesson.md` בחבילת ההוראה (ראו הפרומפט הנפרד ליצירת תוכן) כחומר שהסטודנט קורא לבד, לא נעלמים מהקורס.
* אף נושא לא נמחק לגמרי — רק שונה הערוץ (הרצאה חיה מול קריאה עצמאית) לפי כמה קריטי הוא לתרגול של אותו שבוע.

## מטרות הקורס

בסיום הקורס הסטודנט יוכל:

* להשתמש ב-AI לפיתוח תוכנה מקצועי.
* לתכנן מערכת לפני כתיבת קוד.
* לנהל Context בפרויקטים אמיתיים.
* לעבוד עם Codebase קיים.
* להשתמש ב-AI Agents לפיתוח.
* לפתח אפליקציית Web מלאה.
* לבצע Debugging ובדיקות באמצעות AI.
* לפרוס מערכת לענן.
* לעבוד לפי Best Practices של AI Software Engineering.

## פרויקט הסמסטר

כל זוג סטודנטים מפתח מערכת אחת לאורך הסמסטר. בכל שבוע מוסיפים יכולת חדשה בהתאם לנושא הנלמד. בסוף הקורס מתקבלת מערכת מלאה שרצה בענן.

**הערה לגבי הערכה (להשלמה מול ועדת הוראה):** מומלץ לקבוע מראש משקל בין הגשות שבועיות, פרויקט סמסטריאלי, והצגה סופית, וכן מנגנון להערכת תרומה אישית בתוך הזוג.
