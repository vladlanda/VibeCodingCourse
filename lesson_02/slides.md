# שיעור 2
## AI Development Environment
### Vibe Coding – AI-Native Software Development

בסוף השיעור תדעו לענות על:
- מה התפקיד של VS Code, Git ו-GitHub בזרימת העבודה?
- מה ההבדל בין כלי Autocomplete לכלי Agentic?
- איך פותחים ומנהלים session עם Claude Code בפועל?
- אילו כלי AI חינמיים זמינים כדי להתחיל בלי תקציב?

---

## VS Code, Git ו-GitHub — יסודות

- **VS Code** — סביבת הפיתוח שרוב כלי ה-AI מתחברים אליה
- **Git** — רשת ביטחון: כל שינוי הוא תמונת מצב שאפשר לחזור אליה
- **GitHub** — אחסון מרוחק ושיתוף פעולה (יורחב בשבוע 10)

**כלל אצבע: Commit נקי לפני שנותנים ל-AI לגעת בקוד**

---

## מה הופך כלי ל"אג'נטי"?

**Autocomplete בסיסי**
- משלים שורה/פונקציה בודדת
- אתם מחליטים הכל

**כלי Agentic (Claude Code)**
- מקבל משימה ברמה גבוהה
- קורא קבצים, כותב במספר מקומות, מריץ בדיקות
- שותף לביצוע משימה — לא רק עוזר הקלדה

---

## Claude Code — איך זה עובד בפועל

1. פותחים session בתיקיית הפרויקט
2. נותנים הנחיה ברורה וספציפית
3. הכלי מציג תוכנית / מתחיל לפעול
4. **בודקים כל Diff לפני אישור**
5. ממשיכים לצעד הבא

זה שלבים 4-6 מה-Workflow שלמדנו בשיעור 1

## Best Practices לעבודה עם כלי Agentic (1)

1. הנחיות ברורות וספציפיות
2. צעדים קטנים, לא משימה ענקית אחת
3. בדיקת כל Diff — תמיד, בלי יוצא מן הכלל

---

## Best Practices לעבודה עם כלי Agentic (2)

4. Git כרשת ביטחון — Commit לפני ניסוי משמעותי
5. לא לסמוך בעיוורון — לבדוק פונקציות לא מוכרות
6. איטרציה, לא ניסיון יחיד

## סיכום

1. VS Code + Git + GitHub = השולחן שעליו עובדים
2. Agentic ≠ Autocomplete — זה שותף לביצוע משימה
3. יש נקודת כניסה חינמית — אין תירוץ לא להתחיל
