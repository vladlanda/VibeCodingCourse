# הערות למרצה — שיעור 2: AI Development Environment

תזמון כולל: 90 דק' תוכן + 45 דק' תרגול, מפגש רציף אחד.

## שקף פתיחה (2 דק')
קשרו לשיעור הקודם: "שבוע שעבר ראיתם דמו של 20 דקות. היום בונים את הסביבה שתאפשר לכם לעשות את זה בעצמכם."

## VS Code, Git, GitHub (20 דק')
- אל תלמדו Git לעומק כאן — זו לא מטרת השיעור. המטרה: להבין *למה* Git קריטי כשעובדים עם AI (רשת ביטחון), לא איך להשתמש בכל הפקודות.
- מוקש נפוץ: סטודנטים חושבים ש-Commit הוא "שלב בירוקרטי מיותר". הדגישו: זו הביטוח שהופך אתכם אמיצים יותר בניסויים עם AI, כי תמיד אפשר לחזור אחורה.
- זמן קצר (2-3 דק') להראות מסך הגדרות VS Code + התקנת תוסף בסיסי, לא יותר.

## Autocomplete מול Agentic (חלק מ-30 דק' של Claude Code) (10 דק')
- זו נקודת המפתח הרעיונית של השיעור. אל תמהרו.
- דוגמה חיה טובה: הראו לכיתה שורת קוד עם Autocomplete (שם פונקציה מושלם אוטומטית) לעומת הנחיה כמו "תוסיף לי אימות משתמשים" שמניעה סדרת פעולות.

## Claude Code בפועל (20 דק')
- עברו על 5 השלבים בשקף בקצב איטי — זה בדיוק ההמשך הישיר של ה-Workflow משיעור 1.
- שאלת דיון: "מה קורה אם מדלגים על שלב הבדיקת ה-Diff?" — תנו לכיתה לענות לפני שאתם עונים.

## כלים חינמיים (15 דק')
- הדגישו: **בדקו את הטבלה הזו מול המצב העדכני בפועל לפני ההוראה** — המידע כאן נכון ליולי 2026 ועלול להשתנות.
- אם יש סטודנטים שכבר יש להם Student Pack מאושר — תנו להם לשתף חוויה קצרה, זה מוסיף אמינות.

## Best Practices (15 דק', שני שקפים)
- אל תקריאו רשימה — לכל כלל תנו דוגמה קצרה של "מה קורה כשלא עוקבים אחריו" (למשל: הנחיה עמומה → AI מנחש ומפספס; דילוג על Diff → קוד שבור נכנס לפרויקט בלי שידעתם).

## הדגמה חיה (עד 25 דק', בתוך ה-30 שהוקצו ל-Claude Code)
ראו demo/README.md ו-demo/prompt.md. השתמשו בפרויקט משיעור 1 כדי ליצור המשכיות ותחושת "אנחנו בונים על מה שכבר יש לנו".

## מעבר לתרגול
"עכשיו תעשו את זה בעצמכם, מהתקנה ועד ה-Commit הראשון שלכם."

## תרגול מודרך — 45 דק'
המוקש הכי נפוץ: בעיות Authentication מול GitHub (HTTPS token מול SSH key). הכינו מראש הוראות קצרות לשתי השיטות והפנו אליהן זוגות שנתקעים, כדי לא לבזבז את כל הזמן על זה.

## סיכום
חזרו ל-3 ה-Takeaways, קשרו לשבוע הבא: "היום בנינו את השולחן. שבוע הבא לומדים איך 'מזינים' לכלי את המידע שהוא צריך כדי לעבוד נכון על הפרויקט שלכם — Context Engineering."
