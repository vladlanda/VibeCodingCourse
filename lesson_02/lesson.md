# שיעור 2: AI Development Environment

## VS Code, Git ו-GitHub — יסודות הסביבה

לפני שבונים עם AI, צריך שולחן עבודה. שלושה רכיבים בסיסיים:

- **VS Code** — סביבת הפיתוח (IDE) שרוב כלי ה-AI האג'נטיים מתחברים אליה כתוסף, או פועלים לצידה מהטרמינל.
- **Git** — מערכת ניהול גרסאות. כל שינוי בקוד נשמר כ״תמונת מצב״ שאפשר לחזור אליה. זו רשת הביטחון שלכם כשה-AI כותב קוד — אם הוא הרס משהו, Git מאפשר לחזור אחורה תוך שניות.
- **GitHub** — האחסון המרוחק של הקוד, ומקום שיתוף הפעולה (יורחב בשבוע 10).

הכלל המעשי: **לפני שנותנים ל-AI לגעת בקוד, מוודאים שיש Commit נקי מאחור.** זה לא פרוצדורה בירוקרטית — זו הביטוח שהופך ניסוי-וטעייה עם AI לבטוח.

**נקודה למסלול בינה מלאכותית:** אם כבר עבדתם עם Colab או Kaggle Notebooks, אתם מכירים את הרעיון של "עותק שאפשר לחזור אליו" ברמה בסיסית (Version history של Colab, למשל). Git הוא הגרסה המקצועית והמדויקת של אותו רעיון — במקום "היסטוריה מעורפלת" שהפלטפורמה שומרת בשבילכם, אתם שולטים מתי בדיוק נשמרת "תמונת מצב", ומה בדיוק נכנס אליה. וGitHub הוא הגרסה המקצועית לשיתוף notebook: במקום לשלוח קובץ .ipynb בוואטסאפ, כל הפרויקט חי במקום אחד עם היסטוריה מלאה.

## מה הופך כלי ל״אג'נטי״?

יש הבדל מהותי בין שני סוגי כלים:

- **Autocomplete בסיסי** (כמו הדורות הראשונים של כלי AI לקוד) — משלים שורה או פונקציה בודדת לפי הקשר מקומי. אתם עדיין מקבלים כל החלטה.
- **כלי Agentic** (כמו Claude Code) — מקבל משימה ברמה גבוהה ("תוסיף התחברות עם Google לאפליקציה"), ומבצע בעצמו סדרת פעולות: קורא קבצים רלוונטיים, כותב קוד במספר מקומות, מריץ בדיקות, ולעיתים גם מתקן את עצמו אם משהו נכשל.

זה בדיוק ההבדל בין "עוזר הקלדה" לבין "שותף לביצוע משימה". הקורס הזה עוסק בסוג השני.

**נקודה למסלול בינה מלאכותית:** זה דומה במידה מסוימת למעבר מ-Feature Engineering ידני למודלים מבוססי Deep Learning — גם שם המעבר הוא מ"אני מגדיר כל שלב בעצמי" ל"אני מגדיר את המטרה, והמערכת מבצעת יותר את הדרך אליה". ההבדל הקריטי: גם כשהמערכת "עושה יותר", עדיין אתם קובעים את המטרה, בודקים את התוצאה, ומחליטים אם לקבל אותה — בדיוק כמו שבמודל ML אתם עדיין אחראים על ה-Evaluation, לא רק על ה-Training.

## Claude Code — איך זה עובד בפועל

Claude Code (וכלים דומים) פועלים בדרך כלל בשתי צורות:

1. **מהטרמינל** — פקודה שפותחת session אינטראקטיבי מול תיקיית הפרויקט.
2. **כתוסף בתוך VS Code** — פאנל צד שרואה את אותם קבצים שאתם רואים.

תהליך עבודה טיפוסי: פותחים session בתוך תיקיית הפרויקט → נותנים הנחיה ברורה → הכלי מציג תוכנית או מתחיל לפעול → בודקים כל שינוי (Diff) לפני שמאשרים → ממשיכים לצעד הבא. זה בדיוק החיבור לשלבים 4-6 מה-Workflow שלמדנו בשיעור 1 (תכנון עם AI → מימוש עם AI → Code Review).

## כלים חינמיים להתחלה

לא צריך תקציב כדי להתחיל:

- **GitHub Copilot Free** — מסלול חינמי רחב, ללא צורך באימות סטודנט, זמין ב-VS Code. נקודת הכניסה הפשוטה ביותר.
- **GitHub Student Developer Pack** — למי שמאמת חשבון סטודנט, מקבל הרחבה (בכפוף לזמינות הרשמה, שמשתנה מעת לעת — בדקו סטטוס עדכני).
- **Cursor** — IDE מלא מבוסס AI, עם מסלול חינמי מוגבל (מכסת השלמות בסיסית).

**הערה חשובה:** שוק הכלים והמסלולים החינמיים משתנה מהר מאוד — בשיעור 1 ראינו איך Gemini CLI החינמי נסגר תוך פחות משנה. תמיד כדאי לבדוק סטטוס עדכני קרוב למועד השימוש בפועל.

**נקודה למסלול בינה מלאכותית:** רובכם כבר משתמשים ב-ChatGPT או Claude בצ'אט רגיל לכתיבת קוד עזר למחקר — סקריפט לניקוי דאטה, פונקציית עזר למודל. זה בדיוק נקודת ההתחלה. ההבדל בין זה לבין כלי אג'נטי הוא לא "כלי אחר לגמרי" אלא שדרוג: במקום להעתיק-להדביק קוד מהצ'אט לקובץ בעצמכם, הכלי האג'נטי כותב ישירות לקבצים, קורא את שאר הפרויקט כדי להבין הקשר, ומריץ בדיקות בעצמו. אותה יכולת שאתם כבר סומכים עליה — רק עם פחות חיכוך והרבה יותר הקשר.

## Best Practices לעבודה עם כלי Agentic

שש הרגלים שמבדילים שימוש מקצועי מ"ניסוי מקרי":

1. **הנחיות ברורות וספציפיות** — "תוסיף התחברות" עמום מדי; "תוסיף התחברות עם Google OAuth, עם redirect לדף הבית אחרי הצלחה" ברור.
2. **צעדים קטנים** — משימה גדולה מתפרקת לצעדים; קל יותר לבדוק תוצאה של צעד קטן מאשר של שינוי ענק.
3. **בדיקת כל Diff** — לפני אישור שינוי, קוראים אותו. תמיד.
4. **Git כרשת ביטחון** — Commit לפני כל ניסוי משמעותי.
5. **לא לסמוך בעיוורון** — אם משהו נראה "יותר מדי חכם" או משתמש בפונקציה שלא הכרתם — בודקים שהיא אכן קיימת.
6. **איטרציה, לא ניסיון יחיד** — אם התוצאה הראשונה לא מושלמת, מבקשים תיקון ממוקד, לא מתחילים מאפס.
