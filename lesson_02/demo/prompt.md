# פרומפט להקלדה חיה — שיעור 2

## שלב 1: להראות Autocomplete (לא אג'נטי)

התחילו להקליד בעצמכם בתוך `index.html`, למשל תוך כדי הוספת פונקציה חדשה, ותנו לכלי להשלים שורה אחת. הדגישו: "זה עוזר, אבל אני עדיין מחליט כל דבר."

## שלב 2: להראות שימוש אג'נטי אמיתי

פתחו session אג'נטי מלא (Claude Code / Cursor Agent Mode) על תיקיית `todo_app`, והקלידו:

```
תוסיף לאפליקציית ה-Todo הזו שדה "עדיפות" לכל משימה (נמוכה/בינונית/גבוהה),
עם צבע רקע שונה לכל רמת עדיפות ברשימה. תשמור על אותו סגנון קוד שכבר קיים
בקובץ.
```

עצרו לפני אישור השינוי, והראו לכיתה את ה-Diff שהתקבל: אילו שורות נוספו, אילו השתנו.

## נקודות לדגש בזמן ההדגמה

- שימו לב אם הכלי "קרא" את הקוד הקיים לפני שהוסיף (למשל שמר על מוסכמות השמות הקיימות) — זה ההבדל בין הבנת הקשר לבין הדבקת קוד גנרי.
- אם הכלי מציע לשנות משהו לא קשור (Scope Creep) — זו הזדמנות טובה להראות איך דוחים חלק מהצעה ומאשרים רק את מה שרלוונטי.
