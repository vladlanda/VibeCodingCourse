# תרגול מודרך — שיעור 1 (45 דקות)

## מטרת התרגול

לחוות באופן מעשי את מלוא ה-Workflow שנלמד בהרצאה (גם אם בקנה מידה זעיר): מהבנת דרישה, דרך שימוש ב-AI למימוש, ועד בדיקה שהתוצר עובד — הכל תוך 45 דקות, בזוגות.

## המשימה

בנו **Todo App** בסיסי באמצעות כלי AI (Claude Code / Cursor / GitHub Copilot — לפי מה שהוקם בסביבת העבודה). האפליקציה צריכה לתמוך ב:

1. הוספת משימה חדשה.
2. סימון משימה כהושלמה.
3. מחיקת משימה.
4. הצגת רשימת המשימות הנוכחית.

אין דרישה לשרת Backend בשלב הזה — אפליקציה שרצה בדפדפן (HTML/CSS/JS, עם שמירה מקומית) מספיקה לגמרי.

## איך מתחילים — Prompt לדוגמה

השתמשו בפרומפט הבא כנקודת פתיחה, והתאימו אותו:

```
אני רוצה לבנות Todo App פשוט שרץ בדפדפן, קובץ HTML יחיד עם CSS ו-JS
מוטמעים (בלי צורך בשרת). דרישות:
- טופס להוספת משימה חדשה
- רשימת משימות עם checkbox לסימון "הושלם"
- כפתור מחיקה לכל משימה
- שמירת המצב ב-localStorage כך שהרשימה נשארת אחרי רענון הדף
- עיצוב נקי ופשוט

תכתוב את הקוד המלא בקובץ אחד. תסביר בקצרה מה כל חלק עושה.
```

לאחר קבלת הקוד הראשוני — אל תעצרו שם. נסו לבקש שיפור אחד נוסף (למשל: מיון משימות, ספירת משימות פתוחות, או עיצוב שונה) כדי לתרגל איטרציה על קוד קיים.

## תוצר מצופה בסוף 45 הדקות (Definition of Done)

- [ ] קובץ `index.html` שנפתח בדפדפן ועובד בפועל (לא רק "קוד שנראה טוב").
- [ ] אפשר להוסיף, לסמן כהושלם, ולמחוק משימה — כל השלושה נבדקו בפועל בדפדפן.
- [ ] המשימות נשמרות גם אחרי רענון הדף (localStorage).
- [ ] כל אחד מבני הזוג יודע להסביר מה הקוד עושה, גם אם AI כתב אותו — זה חלק מהתרגול, לא בונוס.

## טיפ

אם ה-AI "ממציא" קוד שלא עובד או משתמש בפונקציה לא קיימת — זה בדיוק המקרה שדיברנו עליו בהרצאה (Hallucination). אל תתפסו את זה כתקלה שלכם — תקנו על ידי בקשה מפורשת מה-AI לתקן, או תיאור מדויק יותר של השגיאה שקיבלתם.
